Architecture


Modules

Angular apps are made up of a collection of one or more modules.
Uses @Modules decorator
Modules are cohesive so they represents a specific feature within the application.
Each module is made up of components, directives, pipes, and services
Each Angular application has at least one root module known as the :
Appmodule

Components

Components are basically classes that interact with the .html file of the component.
Uses @Component decorator
A component controls a portion of the view on the browser
Each Angular application will have at least one component known as the root component with the name AppComponent Component consists of the following files:

  • app.component.css
  • app.component.html
  • app.component.ts
  • app.component.spec.ts

    Directives

    Is the same as a component without a template
    Uses @Directive decorator
    Is a mechanism of attaching behaviour to elements in the DOM
    There are two types of directives which are for DOM manipulation:

    1. Attribute directives
      • Manipulate the DOM by changing its behavior and appearance such as:
      • Conditional style to elements
      • Show or hide elements
    2. Structural directives
      • Manipulate the DOM by creating and destroy DOM elements

    Services

    Angular services are singleton objects that get instantiated only once during the lifetime of an application.
    Servces are responsible for sharing data across multiple components, wrtie logic that doesnt require a view, external interactions ie connect to a database
    You use services via Dependancy injections

    Pipes

    Allow us to tranform the data for the view IT DOES NOT CHANGE THE PROPERTY OF THE ATTRIBUTE
    A pipe takes in a value or values and then returns a value
