Property Binding

There is a difference between an HTML attirbute and DOM properties (but they look the same)
Attributes are defined with HTML, they CANNOT change once initlaize and basically represent the initilized value
Properties are defined via DOM, they CAN change once initlaize and basically represent the current value
Attributes initlaize DOM properties
When we do property binding we are binding to the DOM property not the HTML attribute
In the example below it shows this exact point
<input type="text" value = "hello" > is equal to
If there is no input text within the input text box the attribute and value is hello
If there is input text within the input text box the attribute = hello ; value = <is the new input text>

< input [id]="myId" type ="text" value = "hello" > OR
< input id={{myId}} type ="text" value = "hello" >is equal to
The id property of the input element is equal to the assigned property "myId" which has the value 'testID'
The problem without the square brackets is it doesnt allow for property Binding for non string elements therefore use the square version

The only issue with interpolation is that non-string attributes do not render the correct way for example below both when disabled is initilized even is "true" or "false" the value it always true since it evaulates the presence of a string as true rather than what the string states
 < input disabled="true" [id]="myId" type ="text" value = "hello" >
< input disabled="false" [id]="myId" type ="text" value = "hello" >
<input disabled="{{ '{' }}{{ '{' }}false{{ '}' }}{{ '}' }}" [id]="myId" type ="text" value = "hello"> hhhhhhh

Instead of using double curly braces we use property binding as shown before
 < input [disabled]="false" [id]="myId" type ="text" value = "hello" >
< input [disabled]="true" [id]="myId" type ="text" value = "hello" >

Instead of using hard coded true or false we use a class var to set it
 < input [disabled]=isDisabledtrue [id]="myId" type ="text" value = "hello" >
< input [disabled]=isDisabledfalse [id]="myId" type ="text" value = "hello" >

Here we can get rid of the brackets and use bind-<attribute name>
 < input bind-disabled=isDisabledtrue [id]="myId" type ="text" value = "hello" >
< input bind-disabled=isDisabledfalse [id]="myId" type ="text" value = "hello" >

Class binnding is binding of css classes to an element via binding
You can pass the element a css class via class attribute
 < span class = "text-success" > hello < /span > will print out hello

You can pass the element a css class via bindings
 < span [class] = "successClass"> hello < /span > will print out hello

If we combine the class binding and class attribute in the same html element it will :
< span class = "text-special" [class] = "successClass" > hello < /span > will print out hello

To have a conditional statment weather ot not to run the class you can do the following
 Here we will give the class attribute the css class danger-class which will be applied if the hasError attribute value is true
Since hasError is true then danger-class will be applied
< span [class.danger-class] = "hasError" > hello < /span > will print out hello
< span [class.danger-class] = "hasNoError" > hello < /span > will print out hello

We bind the messageClasses object to the [ngClass] directive and now angular applies the messageClasses to the element and checks which classes are set to true
therefore danger-class and text-special will be set
<span [ngClass] = "messageClasses" > hello < /span > = hello

Event binding allows you to listen for certain events such as keystrokes, mouse movements, clicks, and touches
It calls the specified method in the component that is associted witht that particular event

Here we are binding the the click event of the button the function onClick() and we can see once we click the greeting happens
< button (click)="onClick()" > Greet </ button > {{greeting}}

Two Way binding

Want to make sure that the model and view are synced
ie say you have a form where you need to update your username and password your model or properties should automatically recive those values,
also when there is an update in the model the view should automatically reflect the updated values.
Two way binding allows us to update the property and display of a value for that property
< input [(ngModel)]="nametwo" type="text" > {{nametwo}}

Here when we type something in the input box the property that is updated is the nametwo property
{{nametwo}}
