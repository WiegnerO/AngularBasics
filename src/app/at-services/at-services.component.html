Services


Servces are responsible for sharing data across multiple components, wrtie logic that doesnt require a view, external interactions ie connect to a database

Create a service

To create a service in the CLI write ng g s <name-of-service> and in the directory we have a folder <name-of-service>.service.ts
Initally the serivce does not have and dependancies so this decorator does nothing but it might have depancies in the future therefore have it incase
The classes that use this service DO NOT have the @Injectable decorator because components have the @Component decorator includes the @Injectable decorator within it

@Injectable

Is a decorator which lets the framework know that in the file this service/component might have injected dependancies (ie have a service injected in it)
If in the @Injectable decorator it has the value providedIn: 'root' this means its a singleton class
A singleton service is a service for which only one instance exists in an app
You can make the service a singleton by setting the providedIn property of the @Injectable() to "root"

Register the service

We need to register the service with the angular injector or else it will be a regular class
You have to choose where you register the service we can register it within the employee-list component , employee-detail component , app-compnent or app module
It is best to register the service at the app module lvl bc then all the components of the module can access this service while if we do the app component lvl only is children can access it
In the app.module.ts file include the service in the providers array and import it too
