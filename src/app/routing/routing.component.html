Routing App

  1. In the terminal (in the app folder) create the components you want to route to using the following CLI command :
    ng g c < Compnent name > -it -is
    g = genertae
    c = component
    it = inline template
    is = inline styles
  2. After we need to configure the routes
    in the app-routing,module.ts we have the line :
    const routes: Routes = [];
    routes is a constant which is strognly typed as Routes from the router packages and this will define all the routes of the application and each route is nothing but an object which contians a path within the url and the component to be rendered when we navigate to the correpsonding path
    in this case we need to configure two routes in the application for : deparment-list and employee1-list
    const routes: Routes = [
    path : 'departments' , component : DepartmentListComponent
    path : 'emplayees', component : Employee1ListComponent
    ];
    If we go to either paths we need to display thier respective components
    Both app.module and app-routing.module will contian the same import statments
    in the app-routing.module.ts you can make an array of all the routing components export it and then import it in the app.module
    export const routingComponents = [DepartmentListComponent , Employee1ListComponent]
    In the app.module you can add to the routing module import statament and remove the other two statments
    import ( AppRoutingModule , routingComponents ) from './app-routing.module';
    //import ( DepartmentListComponent ) from './department-list/department-list.component';
    //import ( Employee1ListComponent ) from './employee1-list/employee1-list.component';

    declarations: [
    ...
    routingComponents
    ]
    The directive router-outlet> which is avaible from the router prackage and marks where the router displays a view
    Now if you go to http://localhost:4200/departments it will display the DepartmentListComponent at yhe botom of the page
