Routing App

  1. In the terminal (in the app folder) create the components you want to route to using the following CLI command :
    ng g c < Compnent name > -it -is
    g = genertae
    c = component
    it = inline template
    is = inline styles
  2. After we need to configure the routes
    in the app-routing,module.ts we have the line :
    const routes: Routes = [];
    Routes is a constant which is strognly typed as Routes from the router packages and this will define all the routes of the application and each route is nothing but an object which contians a path within the url and the component to be rendered when we navigate to the correpsonding path
    const routes: Routes = [
    {{ '{' }}path : '', redirectTo : '/architecture', pathMatch : 'full'{{ '}' }},
    {{ '{' }}path : 'architecture' , component : ArchitectureComponent{{ '}' }},
    {{ '{' }}path : 'employeeList' , component : employeeListComponent{{ '}' }}
    {{ '{' }}path : 'employeeList/:id' , component : employeeDetailComponent{{ '}' }}
    ];
    The first object represents the 'home' page when you go on www.AngularBasics.com where the path is empty go to the site it will redirect to the www.AngularBasics.com/architecture and display this architecture page (which is the second object)
    The third object represents a page in the application where if you go to www.AngularBasics.com/employeeList it will display the employeeListComponent
    The fourth object represents a page in the application where if you go to www.AngularBasics.com/employeeList/:id it will display the employeeDetailComponent of a specific employee
    The :id represents a placeholder parameter which will be passed into the component
    in the app-routing.module.ts you can make an array of all the routing components export it and then import it in the app.module
    The directive router-outlet> which is avaible from the router prackage and marks where the router displays a view
    In the app.component.html base html file it contains the tags:
    <router-outlet></router-outlet> which will display the coresponding component when a user requests a specific url
    In the index.html it contians the tags:
    <base href="/" > which represents the base of the application and where the other tags are relative to
