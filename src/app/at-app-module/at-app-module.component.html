app.module.ts


The app.module.ts file consists of three sections:
  • imports
  • @NgModule directive
  • constructor
    •

    imports

    The import section will contain an import of the browser module which contains core directives and pipes.
    import {{ '{' }}BrowserModule{{ '}' }} from '@angular/platform-browser';
    The import section will contain an import of the NgModule Decorator.
    import {{ '{' }}NgModule{{ '}' }} from '@angular/core';
    The import section will also contain an import of all the components used in the application.
    import {{ '{' }}ArchitectureComponent{{ '}' }} from './architecture/architecture.component';

    @NgModule

    NgModule is a decorator
    Takes in an object that has standard (meta data) properites to configure the class:

    • Imports
      • Has the exported declarations of other modules available in the current module
      • These modules can either be supporting modules or custom made modules
        • supporting modules examples are FormsModule, RouterModule, CommonModule
    • Declarations
      • Make directives (including components and pipes) from the current module available to other directives in the current module.
      • Everyone inside declarations knows each other
      • For example:
        • We have a component called UsernameComponent
        • We have a pipe called toupperPipe
        • If we want to use the toupperPipe in the UsernameComponent since UsernameComponent knows that the toupperPipe exists from the delcorations array.
    • Providers
      • make services and values known to DI (dependency injection)
      • They are added to the root scope and they are injected to other services or directives that have them as dependency
