To see if a package is outdate you type

update a package

to update a specific package type:
npm update (package_name) or npm install (package_name)

remove a package

to remove a specific package type:
npm uninstall (package_name)

undertsanding version numbers

X.Y.Z
X = major release
Y = minor release
Z = Patch release
^ = will only install that version but not higher ie ^2.3.6 then it will only get updated until 2.9.9 but never above 2
~ = will only install patches only ie ~2.3.6 then it will only get updated until 2.3.9 but never above 2.3

package-lock.json

This is used when multi ppl use this repo
This ensures that when you run npm install it doesnt update to newer versions of the dependancies
package.json -> "express" : "^4.15.4"
run npm install
package.json -> "express" : "4.25.6"
package.json -> "express" : "^4.15.4"
package-lock.json -> "express" : "4.15.4"
run npm install
package.json -> "express" : "4.15.4"

npm cache

npm keeps a cache of your install modules so that it doesn't have to get them every time
This can lead to unexpected results
When you try to install the module or a version of a module that doesn't work, just try clearing your cache.
Clearing your npm cache should always be part of your troubleshooting steps when working with modules
npm cache verify npm cache clean --force

npm scripts

In the package.json file there is a scripts object here will run the npm scripts
You can use default npm scripts such as test or make your own
The key or left side values ie "test" is a script propety and on the value right side ie "ng serve" is the value when you run the command
If I type npm start it will run ng serve , if I do npm test it will run ng test and so on and so forth
I can make my own script such as "orisnewcommand": "ng serve" where when I do npm orisnewcommand it will do ng serve 

            
              "scripts":
